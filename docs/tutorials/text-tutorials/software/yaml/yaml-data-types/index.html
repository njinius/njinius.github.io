Data Types

Supported Data Types#

Strings#

In YAML, you usually do not have to specify strings in quotes unless you intend to handle escaped sequences.

example1: this is a string
example2: 'this is also a string'
example3: "and so is this\n"

Nulls#

In yaml, you can specify a null value using the ~ character or the null string literal.

example1: ~
example2: null

Boolean#

YAML has several supported Boolean values. The following keyword are all supported Boolean values.

  • True
  • Yes
  • On
  • False
  • No
  • Off
example1: True
example2: On
example3: Yes
example4: False
example5: Off
example6: No

Dictionaries#

The dictionary or key-value pair make up the basic building blocks of a YAML document. Every item in a YAML document is a member of a dictionary. Dictionaries can be defined in two different ways, as shown in the following sub-sections.

Inline dictionaries#

example1: {my-key: value, my-other-key: value2}

Multi-line dictionaries#

example1: value1
example2: value2

Arrays#

In YAML, arrays or lists can be defined in two ways, as can be seen in the sub-sections below.

Inline arrays#

example1: [1,2,3]

Multi-line arrays#

example1:
- 1
- 2
- 3
