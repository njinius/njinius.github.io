Syntax

Basic Syntax#

This section covers the syntax of a YAML file. Let us take a look and dissect the following YAML file.

---
name: “John Doe”
age: 39
active: true
friends:
- Bob
- Jim
- Tim
shopping-list:
beers: 5
fruits:
apples: 3
oranges: 2

The three dashes at the start of the code snippet above indicate the start of a new file. Parsers generally recognise the dashes as the beginning of a new file.

---

Next, we see a common YAML construct, a key-value pair where name is the key, and John Doe is the associated value.

YAML can support several data types. In this example, the first three key-value pairs have different data types. The name key has a string for a value. The age key has an integer for the value, and the active key has an associated Boolean value.

Note: Strings can be enclosed in single or double quotes.

name: “John Doe”
age: 39
active: true

The next item, friends, is an array containing three elements, each indented and denoted by a dash. Indentations are how YAML denotes nesting.

Note: Indentations must be made up of even spaces.

friends:
- Bob
- Jim
- Tim

Next, we see another item, shopping-list, containing additional items within it, indicated by the indentations. This item can be viewed as a dictionary containing an integer and another dictionary.

The dictionary named fruits within the shopping-list item contains two integer key-value pairs.

shopping-list:
beers: 5
fruits:
apples: 3
oranges: 2
