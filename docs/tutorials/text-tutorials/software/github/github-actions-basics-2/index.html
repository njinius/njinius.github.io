GitHub Actions - Part 2

In this section, we cover the basics required to get started with Github actions.

Creating, editing and running workflows#

YAML#

GitHub workflow files are defined in YAML. More information and a guide to using YAML can be found here.

Key components of a workflow file#

Name#

The name of your workflow. The name keyword can also be used to name workflows, jobs and steps.

Example

name: Example Workflow

On#

The Github event that triggers the workflow

Example

on: push

The snippet above triggers the workflow

Jobs#

Workflows consist of one or more jobs. It groups the functionality that runs in the workflow. A job consists of one or more steps containing a sequence of commands to run for that particular job.

Example

jobs:
example-job:
name: Example Job
runs-on: ubuntu-latest
steps:
- name: Check out repository code
uses: actions/checkout@v2
- name: Run my test script
env:
my-var: True
run: |
python ./src/my-test-script.py
  • The runs-on keyword lets you select the OS your workflow should run on.
  • The uses keyword tells the job to retrieve one of the community actions. In this particular case, the action (checkout@v2) checks-out your repository so that your workflow can access it.

Env#

Allows the user to define a map of environment variables that can be accessed and used across all available jobs and steps

Example

env:
my-var: true
