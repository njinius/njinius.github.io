NJinius Challenges

Overview#

When the NJinius community are faced with a technical coding problem, we believe the best approach is to share and solve the problem together. We aim to collaborate and develop tangible solutions that can be shared within the NJinius and broader open-source community. We should strive to work together on an issue and also reap the rewards as a community! Often these problems can vary in difficulty, from a small function to a large and complex part of a cloud-hosted platform. As a community, we share the sentiment "All roads lead to Rome" and as such understand that there may be multiple ways to solve a challenge.

Can a Python script that runs an automatic deployment be the best option?

Can we possibly utilise a cloud service technology like AWS EC2 or Azure Functions for speed and flexibility?

Our goal is to not only encourage discussions but also write the code and deploy the solutions for the broader community. Technology is built on sharing, and that's how we progress further onto bigger and better breakthroughs!

What is an NJinius challenge?#

A technical coding problem that can include a wide array of languages and technologies

Who can create an NJinius challenge?#

Any member of the NJinius community with an authorised Github account

Who can contribute to a challenge solution?#

Any member of the NJinius community with an authorised Github account

Please speak to an administrator in our Slack workspace to approve your account account

How to create a challenge#

Challenges pre-requisite

Must-Have:

An authorised NJinius Slack workspace account. An authorised Github account and a bio entry in the #introductions Slack channel

Please visit the following page to find out more information on how to contribute

  1. Request access to the NJinius Slack workspace.
  2. Visit the #challenges channel
  3. Create a new post prefixed with Challenge: Title of your challenge
  4. Work with the community to discuss the problem and come up with solutions
  5. If we can share the problem(s) and solution(s) easily, then we discuss how to get everything added to Github
  6. If you haven't already, request access to the NJinius Github project
  7. Create a GitHub repository for your challenge in the NJinius GitHub project and add a description of your challenge in the README.md file

We are finalising the challenges area. There is more we would like to resolve relating to the structure and process of engaging in challenges. An announcement will be made in our Slack workspace once the challenges are fully operational.

We're cooking something up, please check back soon!
