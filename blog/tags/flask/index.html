Deploy Flask application(s) to Apache Web Server (Windows)

November 23, 2020 · 4 min read

Elvin White

Head of Core Engineering

I have created a simple guide describing how to set up and configure Apache 2.4 to serve Flask web applications on Windows.

Why?#

I have been working on a small project, building a concept prototype for a web application written in Python using Flask, and one of the requirements was to deploy the app on windows (Don't ask why). It was my first time deploying a python web application and using Apache, for that matter. I found a few sources online explaining how to achieve the objective, and this is my attempt at bringing that all together into a guide for future reference. Still, hopefully, others may also find it useful.

Installation#

I chose to use the Apache HTTP server as it seems to be the most recommended.

Install Apache#

  • Visit https://www.apachelounge.com/download/VC10 and download Apache 2.4
    • Make sure you have the correct version of Visual C++ Redistributable for Visual Studio installed.
  • Extract the files; I recommend extracting or copying into the C drive, e.g. 'C:\Apache24'
  • If you wish to run Apache HTTP Server as a windows service, open up a command prompt in the 'C: \Apache24\bin' directory and run the following command 'httpd.exe –k install'

Install the mod_wsgi module#

Mod_wsgi is an Apache module that can host any Python web application. There are several ways to install the module, including the use of Pythons package installer (PIP), that said; in this guide, we will download the precompiled binary and copy it to the required location.

  • Download the archive via the following link https://github.com/GrahamDumpleton/mod_wsgi/releases/download/4.4.12/mod_wsgi-windows-4.4.12.tar.gz
  • Extract the files to a directory
  • Within the extracted folder, locate the .so file corresponding to the version of Apache and Python that you have installed; the folders and binaries are labelled to reflect this.

For example, if you have the 64-bit version of Apache 2.4 and Python 2.7, you will find a file called 'mod_wsgi-py27-VC9.so' in a folder labelled 'Apache24-win64-VC10'.

  • Copy the 'mod_wsgi-py27-VC9.so' to 'C:\Apache24\modules'
  • Within the 'C:\Apache24\modules' rename the 'mod_wsgi-py27-VC9.so' file to 'mod_wsgi.so'

Configure Apache to run the Python application#

Flask Application#

If you don't already have a Flask application that runs using Flask's built-in web server, you can use the sample code below.

from flask import Flask, request
APP = Flask(__name__)
@APP.route('/hello_world')
def hello_world():
return 'Hello World!!'
if __name__ == '__main__':
APP.run()

Create the following directories 'C:\flask\webapp' and save the code above in a file named 'hellofromflask.py' within the 'webapp' directory. (This can be stored wherever you please as long as you refer to the correct file path where necessary).

Creating WSGI File#

Apache uses the .wsgi file to serve the Flask applications, so we need to create a .wsgi file that invokes our application.

  • Open a text editor, insert the following snippet of code.

    Require all granted
    </Directory>
    WSGIScriptAlias / "C:\flask\webapp\myflaskapp.wsgi"
    </VirtualHost>

  • Save the file. I have saved the file at 'C:/flask/webapp/config/my-vhosts.conf'

In the configuration above, ServerName is the domain name for the application. WSGIScriptAlias specifies the script that will run when the Apache Server gets a request. Directory < directory path> enables us to apply directory access permissions for the WSGI script. To configure the application,

  • Open the default configuration file (httpd.conf) in a text editor.
  • Enable the WSGI module by adding the following line to the config file (ideally in the region where other modules are loaded, search '# Dynamic Shared Object (DSO) Support' to find the section)
    ```
    LoadModule wsgi_module modules/mod_wsgi.so
    ```
  • Include your virtual host configuration by adding the following line to the httpd.conf file (ideally under the Virtual hosts' section, search for '# Virtual hosts' to find that section)
    ```
    Include C:/flask/webapp/config/my-vhosts.conf
    ```
  • Save the file.

If you are using the existing vhost config file, instead of the steps above, search for the line, "Include conf/extra/httpd-vhosts.conf", make sure it exists and is not commented out (a comment contains the '#' character prefix).

Start apache (or restart if already running), point your web browser to 'localhost/hello_world' (or whichever URL you are using for your app), and you should see the web app's content displayed.

There you have it. That's pretty much it! Adios.
